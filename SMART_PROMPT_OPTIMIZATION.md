# 智能提示词优化报告

## 📋 优化背景

用户反馈：
> "加了标题以后，提示词得优化，因为有些仅仅是标题能抽取出需求（长标题或者包含需求），而有些不能（简短标题或者不包含需求），所以需要设计提示词去智能判断"

---

## 🎯 问题分析

### 两种典型场景

#### 场景1：长标题/详细标题（包含具体需求）
```
## 系统需支持1000个并发用户，响应时间不超过2秒
```
- ✅ 标题本身就是完整需求
- ✅ 应该从标题中提取
- ✅ original_text应填写标题内容

#### 场景2：短标题/分类标题（不包含需求）
```
## 性能要求

系统应支持不少于1000个并发用户...
```
- ❌ 标题只是分类标签
- ✅ 需求在原文中
- ✅ original_text应填写原文内容

#### 场景3：标题和原文都有需求
```
## 系统需采用B/S架构

前端需支持Chrome、Firefox、Safari等主流浏览器...
```
- ✅ 标题包含一个需求
- ✅ 原文包含另一个需求
- ✅ 应该提取为两条需求

---

## ✅ 优化方案

### 核心策略：智能识别 + 示例引导

在提示词中添加三层指导：

### 1. 策略说明（告诉LLM如何判断）

```markdown
**智能识别标题中的需求**：
1. **长标题/详细标题**：如果标题本身包含具体需求，请从标题中提取需求
2. **短标题/分类标题**：如果标题只是分类名称，则从原文中提取需求
3. **综合情况**：标题和原文都可能包含需求，请两者都仔细分析

**提取来源判断**：
- 标题包含明确需求 → 从标题提取，original_text填写标题内容
- 标题仅是分类 → 从原文提取，original_text填写原文内容
- 标题和原文都有需求 → 分别提取，各自记录original_text
```

### 2. 规则明确（告诉LLM提取要点）

```markdown
3. **提取要点**：
   - requirement：用简洁的语言概括需求（1-2句话）
   - original_text：**从标题或原文中精确摘录**（根据需求来源）
   - page_number：需求所在的页码
   - response_suggestion：建议的应答方向（1句话）
   - risk_warning：潜在风险提示（如果有，没有则填"无"）
   - notes：其他备注（如果有，没有则填"无"）
```

### 3. 示例演示（告诉LLM具体怎么做）

#### 示例1：长标题包含具体需求
```markdown
**输入**：
## 系统需支持1000个并发用户，响应时间不超过2秒
（原文为空或仅有补充说明）

**输出**：
- requirement: "系统需支持1000个并发用户，响应时间不超过2秒"
- original_text: "系统需支持1000个并发用户，响应时间不超过2秒"（从标题提取）
- notes: "关键性能指标，来自标题"
```

#### 示例2：短标题，需求在原文中
```markdown
**输入**：
## 性能要求
系统应支持不少于1000个并发用户同时在线，响应时间不超过2秒。

**输出**：
- requirement: "系统需支持1000个并发用户，响应时间不超过2秒"
- original_text: "系统应支持不少于1000个并发用户同时在线，响应时间不超过2秒。"（从原文提取）
- notes: "关键性能指标"
```

#### 示例3：标题和原文都有需求
```markdown
**输入**：
## 系统需采用B/S架构
前端需支持Chrome、Firefox、Safari等主流浏览器，后端需支持分布式部署。

**输出**（两条需求）：
1. requirement: "系统需采用B/S架构"
   original_text: "系统需采用B/S架构"（从标题提取）

2. requirement: "前端支持主流浏览器，后端支持分布式部署"
   original_text: "前端需支持Chrome、Firefox、Safari等主流浏览器，后端需支持分布式部署。"（从原文提取）
```

## 📝 完整优化后的提示词

### 核心结构

```python
def _build_extraction_prompt(node: PageIndexNode, content: str) -> str:
    prompt = f"""
你是招标文件分析专家。请分析以下章节内容（包含标题和原文），提取所有招标需求。

## 章节信息
- 页码范围：{node.start_index}-{node.end_index}
- 节点ID：{node.node_id or "UNKNOWN"}

## 章节内容（标题+原文）
{content}

## 提取策略（重要）

**智能识别标题中的需求**：
1. **长标题/详细标题**：如果标题本身包含具体需求，请从标题中提取需求
2. **短标题/分类标题**：如果标题只是分类名称，则从原文中提取需求
3. **综合情况**：标题和原文都可能包含需求，请两者都仔细分析，避免遗漏

**提取来源判断**：
- 标题包含明确需求 → 从标题提取，original_text填写标题内容
- 标题仅是分类 → 从原文提取，original_text填写原文内容
- 标题和原文都有需求 → 分别提取，各自记录original_text

## 提取规则
[需求范围、排除内容、提取要点...]

## 示例说明
[3个具体示例...]
"""
```

## 🔍 技术要点

### 1. Few-Shot Learning
使用3个示例演示不同场景，让LLM通过示例学习：
- 示例1 → 理解长标题处理
- 示例2 → 理解短标题处理
- 示例3 → 理解综合处理

### 2. 明确的判断标准
```
判断标准：
- 标题是否包含具体数字、指标、技术词汇？
- 标题是否表达完整的需求语句？
- 标题是否只是分类/章节名称？
```

### 3. 灵活的提取策略
```python
# 不是硬编码规则，而是让LLM智能判断
if 标题包含具体需求:
    从标题提取
elif 标题只是分类:
    从原文提取
else:  # 综合情况
    两者都分析
```
