# 提示词输出优化 - 杜绝思考过程

## 问题现象

LLM在摘抄原文时输出了思考过程和分析，而不是直接输出纯文本：

```
正文: 根据规则，需找到"§2.1.3.1.权限分配"标题之后、"§2.1.3.2.权限控制"标题之前的内容。

在文本中定位：
- "§2.1.3.1.\n权限分配" 出现在第4页末尾；
- 其后内容为：
...

严格逐字复制，保留原文换行，不加空行、不加解释。

输出如下：  
```
根据用户角色和职责，合理分配操作权限。
支持权限的动态调整，以适应业务需求变化。
```
```

## 根本原因分析

### 1. 原提示词结构问题
- **过于详细的步骤说明** → 引导LLM展示思考过程
- **复杂的规则列表** → 让LLM觉得需要"展示理解"
- **示例中的代码块** → 引导输出格式化内容

### 2. 心理学角度
当提示词说"操作步骤"、"找到标题"、"开始复制"时，LLM会倾向于：
- 展示它如何执行这些步骤
- 解释它在做什么
- 证明它理解了任务

这是因为训练数据中，详细的任务描述通常伴随详细的执行过程。

## 优化策略

### 核心原则：命令式 > 描述式

❌ **旧提示词（描述式）**：
```
🔍 **操作步骤**：
1. 在下面的文本中找到"{node_title}"标题
2. 从标题**之后的第一行**开始复制
3. {boundary_instruction}
4. 不包含标题本身
```

✅ **新提示词（命令式）**：
```
逐字摘抄PDF文本中"{node_title}"标题之后的内容，{boundary_desc}。

【严格规则】
1. 直接输出纯文本，不要输出任何思考过程、分析、解释或步骤
```

### 关键变化

| 方面 | 旧版本 | 新版本 | 效果 |
|------|--------|--------|------|
| **指令风格** | 描述式步骤 | 直接命令 | 减少思考输出 |
| **规则数量** | 5条通用规则 + 4条步骤 | 7条核心规则 | 更聚焦 |
| **格式标记** | 使用代码块、emoji | 纯文本 + 中文方括号 | 避免格式化 |
| **示例展示** | 带代码块的示例 | 纯文本示例 | 直接示范 |
| **明确性** | 间接提及 | **直接说明"不要思考过程"** | 最关键！ |

## 优化后的提示词特点

### 1. **第一句就是命令**
```
逐字摘抄PDF文本中"{node_title}"标题之后的内容
```
- 直接告诉LLM要做什么
- 不解释为什么，不描述如何

### 2. **明确禁止思考输出**
```
1. 直接输出纯文本，不要输出任何思考过程、分析、解释或步骤
```
- **这是最关键的一条**
- 直接禁止LLM输出思考过程

### 3. **极简化规则**
- 从9条规则 → 7条规则
- 删除所有"操作步骤"
- 只保留核心约束

### 4. **纯文本示例**
```
【示例】
PDF文本：
§2.1 安全要求
系统需支持SSL加密。
支持双因素认证。
§2.2 性能要求

提取"§2.1 安全要求"应输出：
系统需支持SSL加密。
支持双因素认证。

（直接输出以上两行，不要"正文："、"输出："等任何前缀或解释）
```
- 不使用代码块（避免引导LLM输出```）
- 示例中明确说明"不要前缀"

### 5. **最后一句是行动指令**
```
【立即输出摘抄结果】
```
- 不说"开始摘抄"（暗示过程）
- 说"立即输出"（强调结果）

## 提示词设计原则总结

基于这次优化，总结出LLM提示词设计的核心原则：

### 1. **命令式 > 描述式**
- ❌ "请执行以下步骤..."
- ✅ "逐字摘抄..."

### 2. **结果导向 > 过程导向**
- ❌ "操作步骤：1. 找到标题 2. 开始复制..."
- ✅ "直接输出纯文本"

### 3. **明确禁止 > 暗示期望**
- ❌ "只输出原文内容"（暗示）
- ✅ "不要输出任何思考过程、分析、解释或步骤"（明确禁止）

### 4. **纯文本 > 格式化**
- ❌ 使用大量代码块、emoji、格式标记
- ✅ 简洁的中文方括号【】和纯文本

### 5. **示例要纯粹**
- ❌ 示例中带注释、解释、格式标记
- ✅ 示例就是最终输出的样子

### 6. **最后一句是行动指令**
- ❌ "开始工作吧"（暗示过程）
- ✅ "立即输出结果"（强调结果）

## 适用场景

这种极简命令式提示词特别适合：
- ✅ 需要精确输出，不要任何解释
- ✅ 文本提取、转换等纯机械任务
- ✅ 对输出格式有严格要求的场景

不适合：
- ❌ 需要推理和分析的任务
- ❌ 需要展示思维过程的教学场景
- ❌ 需要多步骤规划的复杂任务
